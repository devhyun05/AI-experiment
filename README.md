# AI Prompt Evaluation Repository

이 저장소는 **AI 프롬프트 실험을 위한 개인 평가 저장소**입니다.  
프롬프트를 단순히 기록하는 것이 아니라,  
**어떤 프롬프트가 더 잘 작동하는지 실험하고 평가하고 개선하기 위한 목적**으로 사용합니다.

---

## 평가 항목

### 1. 프롬프트 품질

프롬프트 품질은 AI가 요청을 **쉽게 이해하고 실행할 수 있도록 작성되었는지**를 평가하는 항목입니다.

좋은 프롬프트는 단순히 길거나 자세한 프롬프트가 아니라,  
**목적이 분명하고, 조건이 명확하며, 출력 형태가 구체적으로 정의된 프롬프트**입니다.

#### 평가 기준

- 목표가 분명한가
- 조건이 포함되어 있는가
- 출력 형식이 지정되어 있는가
- 모호한 표현이 적은가
- 불필요한 정보가 과도하지 않은가

#### 점수 기준

| 점수 | 설명 |
|------|------|
| 1점 | 요청이 매우 막연하고 목적이 불분명함 |
| 2점 | 목표는 있으나 조건과 형식이 거의 없음 |
| 3점 | 기본 목적은 전달되지만 세부 조건이 부족함 |
| 4점 | 목적, 조건, 형식이 비교적 명확함 |
| 5점 | 목적, 조건, 출력 형식이 모두 명확하고 바로 실행 가능한 수준임 |

#### 예시

- 낮은 품질의 프롬프트  
  - `마케팅 문구 써줘`

- 높은 품질의 프롬프트  
  - `20대 직장인을 대상으로 한 인스타그램 광고 문구를 작성해줘. 제품은 생산성 앱이고, 톤은 친근하고 가볍게, 결과는 3줄로 작성해줘.`

---

### 2. 결과 일관성

결과 일관성은 비슷한 프롬프트를 반복했을 때 **비슷한 수준의 품질과 형식이 유지되는지**를 평가하는 항목입니다.

AI를 잘 활용한다는 것은 한 번 좋은 결과를 얻는 것이 아니라,  
**원하는 품질의 결과를 반복해서 안정적으로 얻는 것**에 가깝습니다.

#### 평가 기준

- 같은 프롬프트에서 형식이 유지되는가
- 결과의 품질 편차가 크지 않은가
- 반복 실행해도 의도한 톤과 구조가 유지되는가
- 다른 작업에도 재사용 가능한가

#### 점수 기준

| 점수 | 설명 |
|------|------|
| 1점 | 결과 편차가 매우 크고 예측이 어려움 |
| 2점 | 일부는 맞지만 형식이나 품질이 자주 흔들림 |
| 3점 | 기본적인 결과는 유지되지만 안정감이 부족함 |
| 4점 | 대부분 원하는 형식과 품질이 유지됨 |
| 5점 | 반복 실행해도 안정적으로 같은 수준의 결과가 나옴 |

#### 예시

- 같은 프롬프트를 5번 실행했을 때
  - 1~2회는 잘 맞고 나머지는 형식이 흔들리면 낮은 점수
  - 5회 모두 비슷한 톤과 구조를 유지하면 높은 점수

### 3. 이해 가능성

이해 가능성은 AI가 생성한 결과물을 **내가 읽고 이해하고 설명하고 수정할 수 있는지**를 평가하는 항목입니다.

특히 코드, 자동화 스크립트, 복잡한 문서 초안처럼  
결과물을 이후에도 계속 활용해야 하는 경우 중요합니다.

좋은 결과물은 단지 정답처럼 보이는 결과가 아니라,  
**내가 직접 다룰 수 있는 결과물**이어야 합니다.

#### 평가 기준

- 결과를 읽고 의미를 설명할 수 있는가
- 구조와 흐름을 이해할 수 있는가
- 작은 수정 정도는 직접 가능한가
- 변수명, 함수명, 문장 구조가 이해하기 쉬운가

#### 점수 기준

| 점수 | 설명 |
|------|------|
| 1점 | 결과가 복잡하고 거의 이해할 수 없음 |
| 2점 | 일부는 보이지만 전체 흐름을 설명하기 어려움 |
| 3점 | 대체로 이해되지만 직접 수정하기는 부담스러움 |
| 4점 | 구조를 이해할 수 있고 간단한 수정도 가능함 |
| 5점 | 전체 흐름을 설명할 수 있고 직접 수정·확장 가능함 |

#### 예시

- 낮은 이해 가능성  
  - 동작은 하지만 변수명과 구조가 복잡해서 왜 그렇게 작성됐는지 설명하기 어려운 코드

- 높은 이해 가능성  
  - 함수 역할이 분명하고, 변수명이 읽기 쉽고, 내가 직접 수정할 수 있는 코드

---

### 4. 검토·개선 능력

검토·개선 능력은 AI의 첫 결과물을 그대로 쓰지 않고,  
**문제점을 찾아 더 나은 방향으로 다시 요청하고 다듬을 수 있는지**를 평가하는 항목입니다.

이 항목은 결과물을 이해하는 것에서 한 단계 더 나아가,  
**품질을 높이기 위한 판단과 피드백 능력**을 본다는 점에서 의미가 있습니다.

#### 평가 기준

- 결과의 오류나 부족한 점을 발견할 수 있는가
- 원하는 방향으로 수정 요청을 할 수 있는가
- 초안에서 최종본으로 품질을 높일 수 있는가
- AI의 답변을 그대로 신뢰하지 않고 검토하는가

#### 점수 기준

| 점수 | 설명 |
|------|------|
| 1점 | 첫 결과물을 거의 그대로 사용함 |
| 2점 | 어색함은 느끼지만 어떻게 고쳐야 할지 말하기 어려움 |
| 3점 | 일부 수정 요청은 가능하지만 개선 방향이 제한적임 |
| 4점 | 문제를 짚고 원하는 방향으로 다시 요청할 수 있음 |
| 5점 | 결과를 검토하고 반복 개선하여 최종 품질을 높일 수 있음 |

#### 예시

- 낮은 검토·개선 능력  
  - 결과가 틀리거나 어색해도 그대로 사용함

- 높은 검토·개선 능력  
  - `예시는 좋은데 문장이 너무 길어요. 10대 대상에 맞게 더 짧고 직관적으로 다시 써주세요.`처럼 구체적으로 피드백함

---

## 평가 방식

각 항목은 1점에서 5점까지 평가합니다.

| 항목 | 점수 범위 |
|------|-----------|
| 프롬프트 품질 | 1~5점 |
| 결과 일관성 | 1~5점 |
| 이해 가능성 | 1~5점 |
| 검토·개선 능력 | 1~5점 |

총점은 **20점 만점**입니다.

### 총점 해석 예시

| 총점 | 해석 |
|------|------|
| 16~20점 | AI를 비교적 잘 활용하고 있음 |
| 11~15점 | 실무 활용은 가능하지만 안정화가 더 필요함 |
| 6~10점 | 결과는 얻지만 통제력은 아직 약함 |
| 1~5점 | 아직 실험과 연습이 더 필요한 단계 |

---

## 작성 원칙

이 저장소에서는 아래 원칙으로 프롬프트를 평가합니다.

1. 막연한 요청보다 구체적인 요청을 우선한다.
2. 출력 형식을 가능한 한 명시한다.
3. 한 번 잘 나온 결과보다 반복 가능한 결과를 더 높게 평가한다.
4. 결과물이 좋아 보여도 내가 이해하지 못하면 낮게 평가한다.
5. 첫 결과물보다 최종적으로 얼마나 개선했는지도 함께 본다.
